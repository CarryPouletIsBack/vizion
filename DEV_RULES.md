# Dev Rules — VZION

## Rôles
- Tu es le dev, je suis le product designer.

## Qualité & vérification
- Je compare le résultat pour éviter les erreurs.
- Code propre, sans `!important`.
- Commentaires en français quand c’est utile.
- Code scalable (structure claire, composants réutilisables).
- À noter : `npm install` remonte 1 vulnérabilité modérée (non forcée).

## Front-end
- UI sombre immersive, desktop-first.
- Variables/tokens Figma à respecter.

## Git
- Ne push que quand je dis de push.
